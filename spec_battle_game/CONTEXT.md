# SmartPhoneSpecBattle - CONTEXT

## AIエージェント間の役割分担 (絶対ルール)
SmartPhoneSpecBattleの開発においては、以下の2つのAIアシスタントが協調して作業を行います。各AIは自身の役割を逸脱してはなりません。

### 1. Antigravity
**担当領域**: フロントエンド・UI/UX・テスト環境
- GUI操作全般
- ブラウザやモバイルエミュレータを用いた結合テスト・画面確認
- Artifacts（画像生成、UIの微調整、モックアップ作成）
- ユーザーに視覚的なフィードバックが必要なタスク

### 2. Claude Code
**担当領域**: バックエンド・ロジック・サーバー環境
- ターミナル操作全般
- 複雑なバックエンド処理やゲームロジック（バトル計算式、ドメインモデル等）の構築・修正
- VPSやモバイル環境でのCUIベースの開発とデプロイ
- Git操作・シェルスクリプト実行など

## 連携フロー
両AIは、非同期的にプロジェクトにアクセスするため、以下のフローを必ず遵守すること。
1. **作業開始時**: 必ずこの `CONTEXT.md` と `docs/TODO.md` を読み込み、現在の状況と制約を把握する。
2. **作業終了時**: 自身の行った変更と、次に他方が行うべきタスクを `docs/TODO.md` に更新して引き継働を完了する。

## 検証手段（完了条件）
コードの追加や変更を行った後、タスクを「完了」とする前に、以下のコマンドを実行しエラーゼロであることを確認すること。
- **静的解析**: `flutter analyze`
- **ユニットテスト**: `flutter test`
両方を通過して初めて、機能実装完了とみなすこと。
