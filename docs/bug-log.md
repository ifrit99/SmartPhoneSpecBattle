# バグログ（Bug Log）

実際に発生した不具合と、その対応・再発防止策を残す。
軽微な typo や一次的に解決した問題は記録不要。「同じ失敗を繰り返さないため」のログに徹する。

---

## テンプレート

```
## YYYY-MM-DD: <タイトル>

- **症状**: ユーザー/開発者から見て何が起きたか
- **影響**: どの画面・機能・ユーザーに影響したか、深刻度
- **原因**: 根本原因（表面的な現象ではなく）
- **対応**: どう修正したか（コミット/PR番号があれば添える）
- **再発防止**: 同種バグを今後防ぐためのチェック項目・テスト・ルール
```

### 症状見出しのサンプル（共有URL生成で不正URL混入）

- **症状**: 共有リンク経由でアプリを開いた状態から再度URLを生成すると、`?battle=...` が二重付与されるケースがあった。
- **影響**: フレンド対戦の共有URLが不正となり、デコード失敗。
- **原因**: `Uri.base.origin` ベースで生成していたため、既存のquery/fragmentが混入。
- **対応**: `Uri.base.replace(query: '', fragment: '')` でクリーンなベースに揃える（PR #11）。
- **再発防止**: URL生成ロジックは必ずテストで「query付き状態から呼んでも純粋なURLを返す」ことを確認する。
