# ガチャ機能 ブラウザテストケース

**対象ブランチ**: `feature/phase4-gacha`
**担当**: Antigravity（ブラウザ/エミュレータ上での結合テスト）
**前提**: Flutter Web ビルドをブラウザで実行し、手動またはブラウザ自動化で検証する

---

## テスト対象画面

| 画面 | ファイル | 概要 |
|---|---|---|
| ホーム画面 | `home_screen.dart` | コイン表示、Gacha/Partyボタン |
| ガチャ画面 | `gacha_screen.dart` | 単発/10連ガチャ実行 |
| インベントリ画面 | `inventory_screen.dart` | 所持キャラ一覧、装備切替 |

---

## TC-1: ガチャ画面への遷移

| 項目 | 内容 |
|---|---|

## TC-2: 単発ガチャ（コイン十分）

| 項目 | 内容 |
|---|---|
| **前提条件** | コイン残高 >= 100 |
| **操作** | 「1回引く」ボタンをタップ |
| **期待結果** | |
| | 1. ボタンが無効化（連打防止） |
| | 2. ガチャ機体が左右に揺れるシェイク演出（約1.5秒） |
| | 3. 効果音が再生される（ボタン音 → スキル音） |
| | 4. 結果ダイアログがスケールアニメーションで表示される |
| | 5. ダイアログにレアリティラベル（N/R/SR/SSR）、ピクセルキャラ、端末名、HP/ATKが表示 |
| | 6. ダイアログ枠の色がレアリティに対応（N=グレー, R=青, SR=金, SSR=紫） |
| | 7. 右上のコイン残高が100減少している |
| | 8. OKボタンでダイアログが閉じる |

## 補足: テスト用データ準備

コイン不足テスト（TC-4, TC-5）を実行するには、DevToolsコンソールから `shared_preferences` のコイン値を直接操作するか、バトルを数回行ってコインを消費しきった状態を作る必要がある。

## 補足: 排出確率テーブル

| レアリティ | 通常排出率 | ステータス補正 | カタログ枚数 |
|---|---|---|---|
| N | 60% | x0.8 | 5端末 |
| R | 25% | x1.0 | 5端末 |
| SR | 10% | x1.2 | 4端末 |
| SSR | 5% | x1.5 | 3端末 |

**10連保証枠**: 9枚通常抽選 + 1枚SR以上保証（SR:67%, SSR:33%）

|---|---|
| 単発ガチャ | 100 |
| 10連ガチャ | 900（10%割引） |
