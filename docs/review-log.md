# レビュー指摘ログ（Review Log）

Codex / Claude Code / 自己レビューで受けた指摘と、その反映・次回への教訓を蓄積する。
同じ指摘を繰り返さないための「自分への申し送り」として使う。

---

## テンプレート

```
## YYYY-MM-DD: <PR番号 or ブランチ名> — <テーマ>

- **指摘**: 何を指摘されたか
- **対応**: 何を修正したか（コミットハッシュがあれば添える）
- **次回の実装前チェック**: 次に似た実装をする時に事前に見るべき点
```

---

## 記録例（既存履歴からの再構成）

### 2026-04-14: fix/review-followups — 50ターン決着ロジックと責務分離

- **指摘**:
  - `BattleEngine` が50ターンで暴走していた（決着判定なし）
  - `ResultScreen` にロジックが集中しすぎている
  - URL/生コード正規化がUI側に散らばっている
- **対応**:
  - `BattleEngine` に「HP割合→絶対値→敵勝利」の3段階判定を追加
  - `BattleResultService` を新設しResultScreenから分離
  - `QrBattleService.normalizeBattleInput` を公開しドメイン層に集約
- **次回の実装前チェック**:
  - 繰り返しループには必ず上限と決着条件を入れる
  - 画面Widgetに「永続化・計算・副作用」を書きたくなったら、まずドメイン層のServiceを検討
  - 入力正規化（trim / URLデコード等）はUIではなくドメイン層のエントリポイントで行う

### 2026-04頃: PR #11 — 共有URL生成

- **指摘**: `Uri.base.origin` だとサブパスが落ちる／既存queryが混入する。
- **対応**: `Uri.base.replace(query: '', fragment: '')` ベースに修正（PR #11）。
- **次回の実装前チェック**: URLを組み立てる時は「現在のURLから派生する」系は必ず query/fragment 扱いを明示する。
