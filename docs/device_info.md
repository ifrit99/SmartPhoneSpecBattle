# デバイス情報取得状況

プレイヤーの実機スペックを取得し、キャラクターのステータスや属性に反映する仕組みの現状をまとめる。

## プラットフォーム別の取得状況

| スペック | Native (Android/iOS/macOS) | Web | 取得方法 |
|---------|---------------------------|-----|---------|
| OS Version | 実値 | `"Web"` 固定 | `Platform.operatingSystemVersion` |
| Device Model | 汎用ラベル (`"Android Device"` 等) | `"Web Browser"` 固定 | `Platform.operatingSystem` 判定 |
| CPU Cores | 実値 | `4` 固定 | `Platform.numberOfProcessors` |
| RAM | CPU コア数から推定 | 同左 | `_estimateRam()` による推定 |
| Storage Free | `32GB` 固定 | `32GB` 固定 | ハードコード |
| Battery Level | `50%` 固定 | `50%` 固定 | ハードコード |
| Screen Size | Widget から取得 | Widget から取得 | `MediaQuery` |

### RAM 推定ロジック (`_estimateRam`)

CPU コア数をもとに以下のルールで推定する（実際の RAM 値は取得していない）:

- 8 コア以上 → 6GB
- 6 コア以上 → 4GB
- 4 コア以上 → 3GB
- 4 コア未満 → 2GB

## ステータスへの反映

| デバイススペック | キャラクターステータス | 計算式 |
|---------------|-------------------|-------|
| RAM (MB) | HP | `((ramMB / 1024) * 20 + 40).round()` → 範囲: 60〜200 |
| CPU Cores | ATK | `(cpuCores * 2 + 2).clamp(8, 25)` → 範囲: 8〜25 |
| Storage Free (GB) | DEF | `((storageFreeGB / 8) + 6).round()` → 範囲: 8〜25 |
| Battery Level (%) | SPD | `((batteryLevel / 10) + 5).round()` → 範囲: 8〜25 |

各ステータスにはさらに **±2 のランダム補正** が加算される。

## 属性（ElementType）の決定

OS Version の文字列から数字部分を抽出し、`数値 % 6` で6属性のいずれかに割り当てる:

| 値 | 属性 |
|---|------|
| 0 | Fire (炎) |
| 1 | Water (水) |
| 2 | Earth (地) |
| 3 | Wind (風) |
| 4 | Light (光) |
| 5 | Dark (闇) |

- 例: Android 14.0 → `14 % 6 = 2` → **Earth**
- 例: iOS 16.0 → `16 % 6 = 4` → **Light**
- Web (`"Web"`) → 数字なし → **Fire**（デフォルト）

## キャラクター名の生成

デバイススペック全体からシード値を生成し、決定的に名前を生成する:

- **構造**: `{属性プレフィックス}・{サフィックス}`
- **プレフィックス**: 属性ごとに4種（例: Fire → フレア / イグニス / ブレイズ / バーン）
- **サフィックス**: 全属性共通8種（ナイト / ガーディアン / ウォリアー 等）
- **決定性**: 同じデバイス → 同じシード → 同じキャラクター名・ステータス

## 制限事項と注意点

- **Web 環境ではほぼ固定値**: CPU コア数 4、RAM 3GB 推定、OS `"Web"` となるため、Web ユーザーのキャラクターは差異が出にくい
- **RAM は実値ではない**: すべてのプラットフォームで CPU コア数からの推定値を使用
- **Storage / Battery は完全固定**: プラットフォームに関わらず `32GB` / `50%` がハードコード
- **ガチャキャラ装備時は実機スペック不使用**: ガチャで入手したキャラクターを装備している場合、プレイヤーの実機スペックは参照されない
