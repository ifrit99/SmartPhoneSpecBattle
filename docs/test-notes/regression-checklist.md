# 回帰チェックリスト（Regression Checklist）

改修後に「最低限これは壊れていないか」確認するための叩き台。
全部必須ではなく、変更箇所に応じて関連項目だけ拾う。

## バトル基本
- [ ] タイトル → ホーム → 「バトル」からCPU対戦を開始できる
- [ ] 属性相性（有利/不利）のダメージ補正が効いている
- [ ] スキル発動時のエフェクト・ログが正しい
- [ ] クリティカル演出・ダメージポップアップが正常に表示される
- [ ] 50ターン到達でも必ず勝敗が決まり、ResultScreenに遷移する
- [ ] リザルト画面で経験値・コイン・図鑑・初回バトル報酬が反映される
- [ ] バトル報酬はCPU対戦のみ付与され、QR/URL対戦では付与されない

## URL共有
- [ ] 「Friend」メニュー → 共有画面でURLが生成される
- [ ] 生成URLが `?battle=...` を1回だけ含み、`=` パディングが除去されている
- [ ] 生成URLをコピー→別タブで開くとゲストキャラで対戦できる
- [ ] URL入力画面でURL全体 / 生コードの両方を受け付ける
- [ ] 改ざんデータ（v2チェックサム不一致）はエラー扱いになる
- [ ] v1形式（旧データ）もデコードできる

## 表示
- [ ] タイトル → ホーム → バトル → リザルト の遷移に破綻がない
- [ ] 図鑑（CollectionScreen）に過去に戦った敵が記録されている
- [ ] ガチャ画面で排出アニメーションが動き、結果が所持に加わる
- [ ] オンボーディング未完了時は初回起動で3ページガイドが出る
- [ ] 初回バトル完了後に次アクション案内ダイアログが出る
- [ ] BGM/SEが再生され、個別ミュートがそれぞれ効く
- [ ] Web版でバッテリーゲージが表示されない（削除済み）
- [ ] 敵キャラのデバイス名に実在ブランドが含まれない

## 既知の注意点
- [ ] `flutter analyze` でエラー/警告ゼロ
- [ ] `flutter test` 全件パス（Mac環境で失敗する場合はDocker: `docker compose exec flutter-dev flutter test`）
- [ ] `flutter build web` が通る
- [ ] デイリー報酬: アプリを一度閉じて日付を跨ぎ、再起動/再前面化した時にポップアップが出る
- [ ] デイリー報酬ポップアップはホーム前面時のみ表示（他画面中は戻ってから表示）
- [ ] Web版デプロイ後、OGP/favicon が意図通り表示される
- [ ] SNS貼り付け時にURL長で切れないこと（Twitter/LINE等で要サンプル確認）
