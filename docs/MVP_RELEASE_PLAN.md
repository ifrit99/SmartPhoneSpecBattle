### v1.1 以降に延期

| 機能 | 延期理由 |
|---|---|
| カメラQRスキャン | カメラ権限で審査リスク増。URLコピペで対戦は成立する |
| ディープリンク受信 | AndroidManifest + ドメイン検証が必要。手入力で代替可能 |
| iOS対応 | Android先行リリース戦略 |
| 動的スペック（デイリー補正） | Phase 5構想。MVP不要 |
| パーツ換装・オーバークロック | ユーザー定着後の収益化要素 |
| AIフレーバーテキスト生成 | 外部API依存。なくてもゲームは成立する |

### MVP版の対戦フロー（カメラ・ディープリンクなし）
```
送信側: ホーム → QR対戦画面 → キャラ選択 → QR表示 + 「URLをコピー」→ SNSに貼る
受信側: URLをコピー → ホーム → 「対戦コード入力」→ ペースト → バトル開始
```

## 2. 技術的な前提条件

### リリース前に対処が必要な項目

| 項目 | 現状 | 対処内容 |
|---|---|---|
| `targetSdk` | 33 | **34以上に更新**（Google Play 新規アプリ要件） |
| `compileSdk` | 33 | 34に更新 |
| リリース署名 | `signingConfigs.debug` | **本番用keystore作成 + build.gradle署名設定** |
| アプリ名 | `spec_battle_game` | ストア向け表示名に変更（例: `SPEC BATTLE`） |
| `applicationId` | `com.specbattle.spec_battle_game` | 必要に応じて変更（リリース後は変更不可） |

### keystore管理の注意
- **keystoreファイルは紛失するとアプリ更新が不可能になる**
- 生成後、安全な場所にバックアップを取ること
- `key.properties` は `.gitignore` に追加し、Gitにコミットしない

## 6. ガイドライン・審査対策メモ

### Google Play 審査で注意すべき点
- **ガチャ機能**: ゲーム内通貨（課金購入不可）のみで回せることをストア説明文にも明記
- **プライバシーポリシー**: デバイス情報（OS、RAM、ストレージ等）を取得する旨を記載
- **コンテンツレーティング**: 暴力表現なし・課金なしのため「全ユーザー対象」想定
- **targetSdkVersion**: 34以上であること（2024年8月以降の要件）

### ストアに必要なアセット一覧
- [ ] アプリアイコン（512x512 PNG）
- [ ] スクリーンショット 最低4枚（16:9推奨）
- [ ] 短い説明（80文字以内）
- [ ] 詳しい説明（4000文字以内）
- [ ] プライバシーポリシーURL
- [ ] フィーチャーグラフィック（1024x500 PNG）— 任意だが推奨
